### 1-1. QR 포스터 — 센터 로고 정밀 보정

**문제**: qr-poster.html의 가운데 로고가 K 박스(텍스트)로만 제공돼서 어플 아이콘으로 바꿀 수 없었음. 바꾼 뒤에도 256 같은 작은 사이즈에서 로고가 우하단으로 쏠려 보임.

**원인 분석**:
- 어플 아이콘(`icon-512.png`) 자체는 시각 중심 거의 잡힘 (offset −2px)
- 진짜 원인은 QRious 라이브러리가 작은 사이즈에서 QR 모듈 픽셀 크기 반올림 잔여 공간을 우하단에 남김 → 캔버스 기하학적 중심에 로고를 박으면 *눈에 보이는 QR* 패턴 기준으로는 우하단 쏠림

**조치**:
1. **신규 자산**: `kwmath/icons/qr-center-logo.png` (1024×1024)
   - `어플로고-removebg-preview.png`에서 alpha 채널로 콘텐츠 bbox 자동 검출 → crop → 흰색으로 리컬러 → 1024×1024 크림슨 둥근 사각(radius 22%) 캔버스 정중앙 배치
   - 콘텐츠 영역 70% 폭으로 적당히 여백
2. **`qr-poster.html` 수정 3곳**:
   - line 116: `_img.src = '/icons/qr-center-logo.png'` (was `/icons/icon-512.png`)
   - `drawCenterLogo()`: 캔버스 중심 대신 `findQrBounds()`로 실제 QR 모듈 bbox 중심에 배치
   - `findQrBounds()` 신규: getImageData로 크림슨 픽셀(#8B1A35) 좌표 스캔해서 minX/maxX/minY/maxY 추출 → 시각 중심 = ((minX+maxX)/2, (minY+maxY)/2)
   - `drawKBox()`: 시그니처 변경 (size 인자 제거, cx/cy 받음) — K 박스도 동일 중심 사용
3. **이전 세션 변경** (5/29 시점):
   - `qr-poster.html` 드롭다운 기본값을 "K 박스"에서 "학원 로고 (어플 아이콘)"으로 변경

**결과**: 어떤 사이즈에서도 시각 중심에 로고가 박힘. 256/512/1024 다 안정.

### 1-3. iOS PWA 상태바 가림 수정 — 14개 HTML

**문제**: 아이폰 PWA에서 노치/다이내믹 아일랜드 영역에 topbar 컨텐츠 (학원 로고·이름 / "내 정보" 버튼·"포털" 버튼)가 가려짐. 안드로이드(관우님 갤럭시 폴드6 포함)는 정상.

**원인**: iOS는 PWA 풀스크린 모드에서 상태바를 *웹 컨텐츠 위에 오버레이* 한다. CSS `env(safe-area-inset-top)` 미사용 시 topbar 상단이 노치에 가려짐.

**조치 (`/tmp/fix_safearea.py` 스크립트로 일괄)**:
1. **viewport-fit=cover 추가** — `<meta name="viewport">` 끝에 ", viewport-fit=cover" 삽입 (이미 있는 3개 페이지 제외, 11개 페이지에 추가)
2. **`.topbar padding 보정** — 2가지 패턴 따로 처리:
   - 표준 패턴 (12개 파일): `padding:14px <h>` → `padding:max(14px,env(safe-area-inset-top,14px)) <h> 14px <h>`
   - admin.html 특수 (height:60px 고정): `padding:0 5%;height:60px` → `padding:env(safe-area-inset-top,0) 5% 0 5%;min-height:60px` (height → min-height로 노치 위로 확장 가능)

**적용 파일 14개**: admin · attendance · install · materials · me · portal · privacy · register · report · reviews · study · terms · test-results · video

**검증**: portal.html / admin.html line 5 (viewport) + line 49/30 (.topbar) 새 패턴 확인. audit.sh 통과 (🚨 항목들은 5/25 이전부터 있던 미해결 — `event`/`this`/`navigator`는 false positive, `closeAddEnrollmentModal` 등은 별도 미완 작업).

**한계 / 추가 작업 가능성**:
- `viewport-fit=cover` 효과: 가로 모드에서 좌우 노치 회피용 padding이 필요한 경우 별도 추가 필요 (현재는 세로 모드 위주이므로 생략)
- 하단 홈 인디케이터(bottom safe-area) 보정: 하단 버튼/링크 있는 페이지에서 필요할 수 있음. 현재 풋터들은 시각적으로 잘림 없어 보임 → 보류

## 4. 핵심 메모 (다음 세션에서 까먹기 쉬운 것)

1. **PPT 빌드 = python-pptx** — npm pptxgenjs는 샌드박스 registry 403. python3 `python-pptx` 1.0.2 사용. `from pptx.dml.color import RGBColor` 주의 (스킬 doc에는 `pptx.dgm.color`로 잘못 적혀있음)
2. **한글 파일명 Write 못 함** — `/sessions/...mnt/outputs/build_pptx_v2.py` 같은 한글 미포함 경로는 Windows 경로(`C:\...\outputs\build_pptx_v2.py`)로 Write 사용. 빌드 산출물의 한글 파일명은 python `prs.save()`로 직접 쓰면 OK.
3. **한글 폰트 샌드박스에 없음** — LibreOffice PDF 변환 시 한글이 □ 박스로 나옴. 레이아웃은 박스 위치로 검증 가능. 실제 PowerPoint에서 열면 정상 렌더링됨. (Nanum/Noto CJK 인스톨 시도했으나 apt sudo 권한 X, GitHub raw도 403)
4. **PPT 자산 경로** — 한글 폴더(`이관우T 성공신화 project/...`) 직접 참조하면 python-pptx가 가끔 깨짐. `outputs/assets/` 영문 폴더에 복사 후 그쪽 참조 권장.
5. **QR 센터 로고 중심 보정 로직** — 캔버스 중심 ≠ 실제 QR 시각 중심. 작은 사이즈일수록 차이 커짐. 픽셀 bbox 스캔이 정답. 이번 수정으로 어떤 사이즈에서도 안정.
6. **qr-poster.html의 K-Box 옵션** — `drawKBox` 시그니처 바뀜 (`size` 빼고 `cx, cy` 추가). 만약 K-Box 모드 다시 손볼 일 있으면 인자 순서 주의.
7. **PPT 색상 팔레트 (재사용)** — 크림슨 #8B1A35, 다크크림슨 #6D1428, 잉크 #1A1A1A, 골드 #C9A77C, 블러시 #F8F0F1, 핑크액센트 #E8C4CB. v3 만들 때 동일 시스템 유지.
8. **PPT 폰트** — 헤더/본문 `맑은 고딕`, 영문/숫자 액센트 `Georgia` italic. Windows 기본 폰트만 사용해서 폰트 누락 위험 0.
