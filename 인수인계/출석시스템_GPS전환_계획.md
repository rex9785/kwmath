# 출석 시스템 GPS 전환 계획
> **현재**: admin에서 학생 카드 옆 [출/지/결] 버튼 수동 클릭 → R2 attendance/{학생}.json 기록 → 학부모 portal `/attendance` 캘린더에서 조회
> **전환 시점**: 학생 30~50명 넘어가서 수동 체크가 부담될 때 (예상 7월 세정학원 정규 운영 후)

---

## 왜 GPS로 가는가

| 현재 (수동) | GPS 자동 (전환 후) |
|---|---|
| 관우T가 매 수업마다 1명씩 클릭 | 학생이 어플로 1번 누르면 끝 |
| 30명 × 1초 = 30초 | 관우T 30초 검토만 |
| 칠판 코드 방식보다 안전 (관우T 직접 확인) | 학원 GPS 반경 안에서만 출석 가능 |

## 시스템 설계

### 흐름
```
[학생 입장]
1. 어플 열기 → 출석 버튼 누름
2. 어플이 자동으로 GPS 좌표 + 시간 서버 전송
3. 서버: 학원 반경 100m 안 + 수업 시작 ±15분 → "출석 대기" 상태

[관우T 입장]
1. 수업 시작 5분 후 admin → "오늘 출석"
2. 출석 대기 학생 리스트 한 번 슥 보기
3. 안 보이는 학생 옆 ❌ (지각/결석으로 변경)
4. 나머지 [전체 확정] → 끝 (30초)
```

### 데이터 모델 (현재와 호환)
- R2 `attendance/{학생}.json` 그대로
- 다만 records 값에 메타 추가:
```json
{
  "name": "홍길동",
  "records": {
    "2026-07-15": {
      "status": "출석",
      "method": "gps",
      "lat": 37.5042,
      "lng": 127.0186,
      "distance_m": 23,
      "checkin_at": "2026-07-15T16:42:30+09:00",
      "confirmed_by_admin": true,
      "confirmed_at": "2026-07-15T16:50:12+09:00"
    }
  }
}
```

### 새 API (계획)
- `POST /api/attendance/checkin` (학생) — 토큰 + GPS 전송 → "대기" 기록
- `GET /api/attendance/pending` (admin) — 오늘 대기 학생 리스트
- `POST /api/attendance/confirm` (admin) — 일괄 확정 + 개별 거부
- 기존 `POST /api/attendance` (수동 입력)는 그대로 유지 (예외 케이스용)

### 학원 위치 등록
- Cloudflare 환경변수에 추가:
  - `ACADEMY_LAT` (예: `37.50421`)
  - `ACADEMY_LNG` (예: `127.01863`)
  - `ACADEMY_RADIUS_M` (예: `100`)
- 또는 학원 여러 곳(그릿에듀, 세정학원)이면 JSON 배열로:
```json
[
  { "name": "그릿에듀", "lat": 37.46xx, "lng": 126.89xx, "radius": 80 },
  { "name": "세정학원", "lat": 37.50xx, "lng": 127.01xx, "radius": 100 }
]
```

### 부정 방지
1. **반경 검증** — 학원에서 멀면 거부 (집에서 누르기 불가)
2. **시간 윈도우** — 수업 시작 ±15분만 인정
3. **관우T 일괄 확정** — GPS 통과해도 관우T가 한 번 본 후에 최종 확정
4. **가짜 GPS 감지** (옵션) — 모의 위치 권한 사용 시 거부 (`ACCESS_MOCK_LOCATION` 안드로이드)
5. **빈도 제한** — 같은 학생이 1시간 이내 재시도 불가

### 학생 UI
- portal에 큰 "📍 출석 체크" 버튼 (오늘 첫 진입 시)
- 누르면 → 위치 권한 요청 → GPS 받음 → 서버 전송 → "출석 대기 중 (관우T 확인 후 확정)" 표시
- 확정 시 푸쉬 알림: "✅ 출석 확정됐어요"

### 관우T UI (admin)
- "📅 오늘 출석" 탭
- 수업 시작 시간대별 그룹 (16:40 / 18:30 / 20:20)
- 각 그룹: 출석 대기 학생 카드 (사진 + 이름 + 체크인 시각 + 거리)
- 한 번 슥 본 후 [전체 확정] 또는 개별 ❌
- 학원에 없는데 GPS 통과한 학생은 빨강 표시 (이상치)

## 전환 작업량 추정
- API 신규/수정: 1.5시간
- 학원 좌표 측정 + 환경변수 등록: 30분
- portal에 GPS 출석 버튼 + 권한 UI: 1시간
- admin에 일괄 확정 탭: 2시간
- 테스트 (실제 학원에서 + 집에서): 1시간
- **합계: 약 6시간**

## 전환 전 준비
1. 학원 정확한 GPS 좌표 측정 (Google Maps 우클릭 → "이 위치는?")
2. 반경 결정 (보통 학원 건물 + 입구 정도면 50~100m)
3. 학부모/학생에게 위치 권한 안내 (어플 처음 사용 시)
