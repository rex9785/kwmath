## HTML 파일 점검

### ✅ index.html — 수정 완료
- **버그 (치명적)**: 전체 사이트 HTML이 `</html>` 이후에 329줄 더 붙어있었음 (구버전 코드가 그대로 append됨)
- 브라우저가 두 개의 사이트를 동시에 렌더링 시도 → 레이아웃 깨짐, 스크립트 중복 실행
- **수정**: Python으로 993번째 줄까지 자르고 `</html>` 재작성 → 994줄 정상본으로 복구

### ✅ admin.html — 수정 완료
- **버그 1 (치명적)**: `showToast()` 함수가 전체 코드에서 호출되지만 정의 자체가 없음 → 공지 추가, 리포트 작성, 파일 업로드, 삭제 등 모든 관리자 액션에서 `ReferenceError` 발생 (화면에는 아무 반응 없음)
- **버그 2 (치명적)**: 학생 목록 탭의 `loadStudents()`, `filterStudents()`, `renderStudents()` 함수 전부 미정의 → 학생 탭 완전히 작동 불가
- **버그 3**: `switchTab()`이 학생/파일 탭 전환 시 데이터를 불러오지 않음
- **수정**: 누락된 함수 4개 전부 추가, switchTab에 lazy loading 트리거 추가

### ✅ report.html (학부모 페이지) — 수정 완료
- **버그 1 (치명적)**: `downloadPdf()`가 `/api/download-file`에 `phone4` 파라미터 없이 호출 → 서버에서 항상 403 Forbidden 반환 → 학부모가 PDF를 다운로드할 수 없음
- **버그 2 (치명적)**: `/api/list-files` 호출 시 인증 토큰 없이 호출 → 서버가 401 반환 → PDF 첨부 파일 목록이 아예 안 보임
- **버그 3**: PDF 파일명으로 `f.name` (존재하지 않는 필드) 사용 → 파일명이 `undefined`로 표시
- **버그 4**: PDF 크기를 `f.size > 1024*1024`로 숫자 비교하는데 `size`가 문자열 → `NaN KB` 표시
- **수정**: 전부 수정 완료 (phone4 파라미터 추가, displayName/sizeLabel 필드 사용)

### ✅ class.html — 정상

## 서버 용량 분석

### Cloudflare 무료 플랜 한도

| 항목 | 무료 한도 | 예상 사용량 | 여유 |
|------|----------|------------|------|
| Pages Functions (Workers) | **하루 10만 요청** | 학부모 100명 × 3요청 = 300/일 | 매우 여유 |
| R2 스토리지 | **10GB** | PDF 자료 수십 개 = 수백MB | 여유 |
| R2 읽기 작업 | **월 100만 건** | 일 수백 건 수준 | 여유 |
| R2 쓰기 작업 | **월 100만 건** | 업로드 주 수십 건 수준 | 여유 |

### Notion API 한도
- **3 req/초 (통합 토큰 기준)**
- 페이지 로드 시 동시 요청이 2-3개 → 동시에 1명씩은 문제없음
- 갑자기 10명이 동시 접속 = 초당 20-30 요청 → **일시적으로 429 오류 가능**
- 단, 에러 핸들링이 모든 API에 구현되어 있어 조용히 실패 (빈 화면, 재시도 메시지 표시)

### 결론
> **현재 규모(학부모 50-150명)에서는 서버 용량 걱정 전혀 없습니다.**  
> 무료 플랜으로 수백 명 규모까지 커버 가능.  
> 트래픽이 폭주하더라도 Cloudflare 자체는 무한 확장되며, 병목은 Notion API 속도 제한뿐입니다.

## 보안 주의사항 (버그는 아니지만 알아두세요)

### ⚠️ phone4 brute-force 가능
- 학부모 전화번호 끝 4자리 = 10,000가지 조합
- `/api/reports?phone4=xxxx` 에 rate limiting 없음 → 이론적으로 자동화 스크립트로 모든 학부모 리포트 열람 가능
- **실용적 위험**: 매우 낮음 (수학학원 대상 해킹 유인 없음)
- **장기적 권고**: Cloudflare의 WAF나 Rate Limiting 룰 추가 고려

### ⚠️ 수강 신청 페이지에 캡챠 없음
- 봇이 자동으로 가짜 수강 신청을 Notion DB에 쌓을 수 있음
- **실용적 위험**: 낮음
- **장기적 권고**: 신청 완료 후 어드민 검토/승인 프로세스 이미 있으므로 큰 문제는 아님

### ⚠️ 어드민 토큰 = 비밀번호
- `admin-auth.js`가 `env.ADMIN_PASSWORD`를 그대로 토큰으로 반환
- 브라우저 Network 탭에서 토큰 노출됨 (HTTPS이므로 외부 도청은 불가)
- **실용적 위험**: 로컬 브라우저에서 개발자 도구 열어야 보임 → 일반 사용자 수준에서는 문제없음
- **장기적 권고**: 별도 세션 토큰 발급 방식으로 교체 가능 (현 규모에선 불필요)
